
# README.md
The files `feature-list.yaml` and `use-case-list.yaml` are not being users right now.
The are place holders templates for a more formal specifications documentation
using structured documentation (e.g. YAML) to capture features and use cases.

## User Stories
User stories keep things simple.
A user stories focus on what the user wants to achieve in a simple, high-level way.
They're short, user-centered narratives that help your team understand what the user wants to do and why.
* [User Stories vs Use Cases - Key Difference & Examples](https://uxcam.com/blog/user-stories-vs-use-cases/)

## Use Cases
Use cases, on the other hand, go deep.
A use case is a tool that helps model user-system interactions.
They lay out each step the system and user take to complete a task—from clicking a button to confirming a transaction.
They're especially useful when you’re dealing with complex flows, edge cases, or integrations.
* [Use cases: The ultimate guide](https://medium.com/@60949161/use-cases-to-model-processes-1fb77d4efb30)
