# Slidev + Excalidraw Presentation Workflow

Slidev is a Markdown-based, browser-rendered presentation framework. You write slides in plain Markdown
and run them with a local dev server. Excalidraw produces the diagrams and slide artwork. The two tools
compose well, but there are three distinct ways to connect them — each with real trade-offs.

---

## Quick-start (skip reading — just run)

```bash
# 1. Install prerequisites (Node ≥ 18, pnpm, Slidev, Playwright)
bash presentations/scripts/setup.sh

# 2. Create a new deck inside presentations/
bash presentations/scripts/new-deck.sh my-talk

# 3. Start the dev server (open browser automatically)
cd presentations/my-talk
pnpm dev

# 4. Export .excalidraw files to SVG when you need crisp images
python3 presentations/scripts/export-diagrams.py --input diagrams/ --output assets/
```

---

## Three options — choose one

### Option A — Direct `.excalidraw` embedding via Slidev addon  ★ recommended

**How it works:** The community addon `slidev-addon-excalidraw` installs a Vue component
`<Excalidraw>` that reads a `.excalidraw` JSON file at runtime and renders it with the real
Excalidraw library — hand-drawn look, correct fonts, everything.

**Pros:**
- No export step; edit the diagram, reload the slide
- Preserves the Excalidraw aesthetic exactly
- Works with your Python-generated `.excalidraw` files from `generate_slides.py`

**Cons:**
- Community addon (not officially maintained by the Slidev team); may lag behind Slidev versions
- PDF export of Excalidraw-rendered slides can be inconsistent
- Requires the `diagrams/*.excalidraw` files to be co-located with your deck

**Slide syntax:**

```markdown
---
addons:
  - excalidraw
---

# My Slide Title

<Excalidraw filePath="./diagrams/architecture.excalidraw" />
```

**Install:**

```bash
pnpm add slidev-addon-excalidraw
```

---

### Option B — Export to SVG, embed as images  ★ most portable

**How it works:** A Playwright + Excalidraw script exports each `.excalidraw` file to an SVG
(or PNG). The SVG is referenced in slides as a regular `<img>`. Slidev handles it like any other
static asset.

**Pros:**
- No addon dependency; works with any Slidev version
- SVG is vector — looks crisp on any screen size or projector
- PDF/PPTX export works perfectly
- The Excalidraw "embed scene" flag in the SVG means the source is recoverable

**Cons:**
- Must re-run the export script when diagrams change
- Requires internet the first time (exports via Excalidraw CDN in a headless browser)

**Slide syntax:**

```markdown
---
# My Slide Title
---

<img src="./assets/architecture.svg" class="h-full" />
```

**Export:**

```bash
python3 presentations/scripts/export-diagrams.py \
    --input diagrams/ \
    --output assets/ \
    --format svg
```

---

### Option C — Use existing PNGs directly  ★ works right now, zero setup

**How it works:** If you already exported PNGs from the Excalidraw desktop app (or from
`generate_slides.py` + `export-diagrams.py`), just reference them in Markdown. No tooling needed.

**Pros:**
- Zero setup — works immediately
- PNGs already exist in `presentations/requirements/`

**Cons:**
- PNGs are raster — they look blurry if the slide canvas is larger than the image resolution
- No round-trip back to Excalidraw source

**Slide syntax:**

```markdown
---
# My Slide Title
---

![](./slide-01.png)
```

## Recommended workflow (combining A + B)

```
generate_slides.py           → output/slides/slide-NN.excalidraw
                                       │
           ┌───────────────────────────┤
           ↓                           ↓
   Option A: reference           Option B: export-diagrams.py
   .excalidraw directly          → assets/slide-NN.svg
   in slides.md via addon        → reference SVG in slides.md
           │                           │
           └───────────┬───────────────┘
                       ↓
                  pnpm dev  (live editing)
                  pnpm build  (static site)
                  pnpm export  (PDF/PPTX for offline sharing)
```

Use **Option A** during development (fast iteration). Switch to **Option B** (SVG) before
the final export to PDF or PPTX for sharing.

## Project structure created by `new-deck.sh`

```
presentations/
└── my-talk/
    ├── slides.md            # your presentation content (Markdown)
    ├── package.json         # Slidev + addon deps
    ├── .gitignore
    ├── diagrams/            # .excalidraw source files (versioned)
    └── assets/              # exported SVG/PNG (versioned, re-generated by script)
```

## Slide anatomy (slides.md)

Slides are separated by `---`. The first slide's YAML block is the global config.

```markdown
---
theme: default
addons:
  - excalidraw          # remove if using Option B or C only
title: My Presentation
aspectRatio: 16/9
---

# Title Slide

Subtitle or opening content

---
layout: center
---

# Diagram Slide — Option A (addon)

<Excalidraw filePath="./diagrams/slide-01.excalidraw" />

---
layout: center
---

# Diagram Slide — Option B (SVG)

<img src="./assets/slide-01.svg" class="h-4/5 mx-auto" />

---

# Slide with notes

Content visible to the audience.

<!-- Speaker notes: only visible in presenter view -->
These are my notes. Not shown on screen.
```

## Presenter mode & Zoom

Slidev has a built-in presenter view with speaker notes:

```bash
# Start dev server with remote access (gives a QR code for audience)
pnpm dev --remote

# Presenter URL (separate browser tab with notes):
# http://localhost:3030/presenter
```

**For Zoom:**
1. Start the dev server: `pnpm dev`
2. Open the presenter view (`/presenter`) on your local monitor
3. Share the main presentation window (`http://localhost:3030`) to Zoom — that URL is the clean
   audience view with no notes
4. Or share your whole screen and use the presenter view locally

**Tip:** Press `F` for fullscreen, `O` for slide overview, `D` for drawing mode (live annotation
on top of any slide without touching your Excalidraw source).

## Export (for sharing or offline backup)

```bash
# Export to PDF (requires playwright-chromium)
pnpm export

# Export to PNG frames (one image per slide)
pnpm export -- --format png

# Export to PPTX (opens in PowerPoint)
pnpm export -- --format pptx

# Build static site (host anywhere, or open dist/index.html offline)
pnpm build
```
